# ai-docs

这个仓库的目的是，使用AI（主要是claude code）以技术栈为中心，构建渐进式的学习文档。

最基本的范式是，在ref-docs中添加某个技术栈的文档目录作为submodule，让AI根据技术栈的官方文档从零开始写新的文档，来教人类快速上手某个技术栈。

本readme供AI阅读，也供人类阅读。

## 学习方式

我喜欢临时决定动态的学习模式，所以我们的流程可以是： 
1. 先确定要学习的技术栈 
2. AI给人类提供集中可选的学习模式和路径，人类选择模式和路径
3. AI生成小篇幅的文档，人类根据文档进行实践，对文档进行补充说明、或者让AI帮忙矫正理解，或者提出文档和实践不一致的部分

现在的问题是：
1. 如何在文档中直观的体现出AI和人类的双重身份？
2. 人类的实践经验和demo代码如何在这个仓库与文档共存，方便以后的自己，或者其他人阅读？

我想到的一个比较好的方法是，让人类通过网页直接对AI内容进行批注和编辑，同时管理技术栈相关的实验代码。系统会把这些交互内容生成为静态网站，这样既保持了丰富的交互体验，又避免了服务器资源消耗。
